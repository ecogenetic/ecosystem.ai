# Local Installation of ecosystem.Ai with Docker

For most scenarios, Docker Compose is the recommended installation method due to its simplicity, ease of use, and reliability.

## Prerequisites

- [`Git`](https://git-scm.com/downloads)
- [`Docker`](https://www.docker.com/products/docker-desktop/)

---

## Installation Steps

Follow these steps to set up ecosystem.Ai with the default configuration:

```bash filename="Clone the Repository"
git clone https://github.com/ecosystemai/install_x86.git
git clone https://github.com/ecosystemai/install_arm.git
```

```bash filename="Navigate to the ecosystem.Ai Directory"
cd install_x86
cd install_arm
```

```bash filename="Create a .env File from .env.example"
cp .env.example .env
```

> **Note:** **If you're using Windows, you might need to use `copy` instead of `cp`.**

```sh filename="Start ecosystem.Ai"
start.bat
```

<Callout type="success" title="Access ecosystem.Ai!" emoji="ðŸŽ‰">
  **Visit [http://localhost:80](http://localhost:80)**
</Callout>

## Update ecosystem.Ai

The following commands will fetch the latest ecosystem.Ai project changes, including any necessary changes to the docker compose files, as well as the latest prebuilt images.

> **Note:** you may need to prefix commands with sudo according to your environment permissions.

```bash filename="Stop the running container(s)""
stop.sh
```

```bash filename="Pull latest project changes"
git pull
```

```bash filename="Pull the latest ecosystem.Ai image""
start.sh
```

```bash filename="Start ecosystem.Ai"
start.sh
```
